# Option Obelisk v1.10.3 — Opal Light Theme + UX Polish

This release adds the app's first light theme and lands a batch of pre-launch
UX polish. No changes to trade math, the Python engine, backend routes, or
stored data. Existing accounts, positions, templates, alerts and preferences
carry over untouched.

## Opal — a light theme

A 4th theme, **Opal**, joins Amethyst, Emerald, and Obsidian in
Settings → Appearance. Light mode: clean near-white surfaces, muted teal
accents, a formal finance aesthetic.

- Built on **Obsidian's** flat, blur-free architecture (the app's cheapest
  theme) rather than the heavier glass themes, so it composites just as fast:
  flat cards with hairline borders, no backdrop blur, no ambient radials, no
  breathing halos. Glow is a `box-shadow` focus ring only.
- `color-scheme: light` is set for Opal, so native controls (inputs, sliders,
  checkboxes, scrollbars, date pickers) render light and aren't force-darkened
  by the OS or a dark-mode extension.
- Every surface, text tier, border, P&L color, chart, input and button
  recolors from the shared theme tokens. Profit renders green, loss red.
  The particle background recolors to teal automatically.
- Theme choice persists per machine, same as the other three.

## UX polish (the v1.10.2 fixes, included here)

1. **Animated launch splash** — the Obelisk insignia fades in and pulses with
   the active theme's accent while the app loads, then crossfades to the app.
   Replaces the bare loading bar. Static under reduced-motion.
2. **Settings tabs scroll** — all nine settings tabs are now reachable; the tab
   strip scrolls horizontally with a themed thin scrollbar instead of clipping.
3. **Snappier tab switching** — settings tab changes are a quick ~200 ms fade
   instead of the old slower slide.
4. **Themed menu icons** — the header ⋮ menu uses monochrome icons that inherit
   the theme color, replacing the colored emoji.
5. **Sign-in insignia** — the sign-in page shows the Obelisk insignia instead
   of the old placeholder shape.
6. **User-friendly error messages** — errors now read as plain, professional
   English. Login failures show a single generic message (no username/password
   enumeration), and no internal or IPC error text reaches the UI. Full errors
   still log to the console for debugging.
7. **Humanized copy** — em dashes removed from all user-facing text across the
   app in favor of periods, commas, and colons.

## Verification

- Frontend `tsc` clean, production build clean, 126/126 backend tests pass.
- Opal verified live across Home, Position Log, Analytics, Screener, Settings
  and the sign-in gate: readable surfaces and borders on white, correct P&L
  colors, `color-scheme: light` confirmed, all four themes switch cleanly
  (only Opal is light), zero console errors.

## Notes / known tradeoffs

- The Opal palette intentionally defines only teal / success-green /
  danger-red plus neutrals. The app's secondary info and warning hues collapse
  to a muted slate (the badge or banner text carries the meaning), the same
  way Obsidian collapses to grayscale. If a distinct warning color is wanted
  later, one amber value would need to be added to the palette.
- This release was developed as v1.10.2 (UX polish) plus v1.10.3 (Opal) and
  ships together as **v1.10.3**.
