# Option Obelisk v1.10.1 — Optimization Release

A focused optimization pass: no new features, no behavior changes for the
user's data or math — just a faster initial load, fewer wasted re-renders,
crash safety, better accessibility, and a cleaner codebase. Every change was
implemented from the v1.9.3 code-optimization audit
(`docs/audits/CODE-OPTIMIZATION-AUDIT-v1.9.3.md`) and verified live in the
browser before shipping.

## Highlights

- **~53% smaller initial load** — the charting library (recharts + d3, 84 KB
  gzipped, measured) no longer loads on the Home screen. It now loads on
  demand only when you open the Analyzer, Sandbox, or Analytics. Initial JS
  dropped from **231 KB → ~109 KB gzipped**.
- **Crash safety** — a rendering error anywhere in the app now shows a
  recoverable panel (Try again / Reload) instead of a blank window.
- **Fewer wasted re-renders** — the eight largest views no longer re-render
  on unrelated 60-second background updates (market pulse, FX, paper marks).
- **Accessibility** — modals now trap keyboard focus and restore it on close,
  a Skip-to-content link was added, and toast confirmations are announced to
  screen readers.

## What changed (from the audit)

1. **Root error boundary** — catches render throws app-wide and offers a
   themed recover/reload panel; your saved data is never touched.
2. **Chart views code-split** — Analyzer, Sandbox and Analytics load their
   chart code lazily (React.lazy + Suspense), cutting the initial bundle by
   ~53%. A separate stable `vendor` chunk (React/zustand) also improves
   cache hits across updates.
3. **Targeted store subscriptions** — the eight big view components now
   subscribe only to the state they use (`useShallow` selectors), so idle
   background writes stop reconciling large subtrees.
4. **Modal focus management** — `role="dialog"`/`aria-modal`, focus moves
   into the dialog on open, Tab/Shift+Tab are trapped inside, and focus
   returns to the trigger on close.
5. **De-duplicated helpers** — the theme-color reader, the P&L color class,
   and the premium/debit basis calculation are now single shared sources
   (previously copy-pasted 3–6×).
6. **Build hygiene** — a stable vendor chunk split; the electron dev
   toolchain lockfile was tidied. (The remaining electron `npm audit`
   findings are dev-only build tooling that never ships in the installer;
   they were intentionally left rather than force a breaking toolchain bump.)
7. **Toast announcements + skip-link** — a persistent `aria-live` region
   announces confirmations; a Skip-to-content link is the first focusable
   element.
8. **Codebase split** — the two largest files were broken up with no
   behavior change: the store's persistence layer moved to
   `lib/storePersist.ts` (store.ts 1087 → 943 LOC), and the Settings tab
   components moved to `SettingsTabs.tsx` (SettingsPanel.tsx 1253 → 123 LOC).

## Not changed

- No user-facing features added or removed.
- No changes to trade math, the Python engine, backend routes, or stored
  data formats. Existing accounts, positions, templates, alerts and
  preferences carry over untouched.

## Verification

- 126/126 backend tests pass; frontend `tsc` clean; production build clean.
- Bundle reduction measured from real `vite build` output.
- Live-verified: error boundary catch + recovery; all views mount after the
  selector conversion (incl. a real screen → analyze → payoff-chart flow);
  modal focus trap in both directions + restore; toast live-region
  announcement; every Settings tab renders after the file split; currency
  read + write round-trip after the store split.
