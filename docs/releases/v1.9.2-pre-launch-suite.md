# Option Obelisk v1.9.2 — Pre-Launch Feature Suite

Eight releases shipped as one program (MASTER-BUILD-DIRECTIVE, July 2026),
taking the app from the v1.7.0 core to launch-ready. Every phase was
built, live-verified in the browser against real data, and committed
separately.

> Versioning note: the directive's publish template referred to the whole
> program as "v1.7.0". The suite's own phase numbering ends at v1.9.2, and
> the app ships as v1.9.2 — the tag matches the app, not the shorthand.

## v1.7.0 — Foundation (renumbered from v1.9.0)
- ETF Asset Screener: 8 filters (price, IV rank, premium, AUM, expense,
  yield, 52-week performance, ATR volatility, theta rank, option volume),
  quick volume bands, primary + secondary sort, configurable/reorderable
  columns, rule-based strategy recommendations.
- CAD currency support: USD / CAD / dual display, daily-cached CAD=X rate
  with fallbacks, historical rate stamping at trade entry and close (no
  invented history on legacy trades).
- Fix: single-symbol quote fetches (yfinance returns MultiIndex columns
  even for one ticker).
- The Settings footer now displays the app version.

## v1.7.1 — Onboarding
- 4-screen tutorial (Welcome → Screener → Analyzer → Position Log &
  Settings) plus a "You're ready" screen; progress bar, Back/Next/Skip.
- Shows once per account, only in the session where the account was
  created; completion stored as `onboardingComplete-<username>`.
- Step 4 embeds live theme + currency quick-settings; replayable anytime
  from Help & Glossary.

## v1.7.2 — Analytics Dashboard + Account settings
- New Analytics view: overview cards (trades, win rate, profit factor,
  gross/net P&L), cumulative-realized-P&L equity curve with per-trade
  tooltips, 8-metric grid, expandable by-strategy breakdown; 30d/3m/6m/all
  ranges and real/sandbox scope.
- Settings → Account: profile facts; password change (current-password
  gated, same strength rules as registration, 10-second sign-out);
  Export All Data (JSON backup incl. preferences), Import with overwrite
  confirmation, Clear All Data; Delete Account (password-confirmed,
  removes registry entry + data directory).

## v1.8.0 — Export/import refinement
- Position Log CSV export with a 22-column picker (new %-gain column;
  selection persists; catalog-ordered files).
- Watchlist CSV export (the metrics the screener actually holds — 52w
  high/low deliberately omitted, those aren't fetched).
- Backup schema carries the account identity; 150-trade import→export
  round-trip covered by tests.

## v1.8.1 — Strategy templates
- Save the current screener strategy + full filter set as a named
  template; Load template… dropdown (user saves + read-only built-in
  presets); usage counts.
- Settings → Templates: rename/description edit, duplicate, delete, JSON
  file export/import (fresh ids on import). Stored per account under
  `strategies:<username>`.

## v1.8.2 — Advanced analytics
- Analytics → Advanced page: Sharpe / Sortino / Calmar over per-trade %
  returns; max/current drawdown, duration, below-high-water-mark chart;
  expectancy, profit factor, recovery factor; MAE/MFE exit-quality
  scatter from observed mark watermarks.
- Edge cases guarded: no losses → Sortino "—" and profit factor "∞";
  fewer than 2 trades or zero variance → "—".

## v1.9.0 — Notifications/alerts
- Local alerts, no external service: daily realized P&L thresholds
  (60-second checks), open-position expiry within 7 days / day-of
  (hourly), Asset Screener IV-rank finds (5-minute checks) — plus
  immediate sweeps when data changes.
- OS notifications via the Web Notification API (auto-granted in the
  packaged app) with in-app toast fallback; per-condition dedup; last 50
  alerts in a viewable history. Settings → Alerts for all of it.

## v1.9.1 — Feedback button
- Header ⋮ menu (Settings / Account / Help & Glossary / Feedback & Bugs).
- Feedback modal: bug / feature / general, 500-char description, optional
  diagnostics and position-log summary (counts only). Sends as a
  pre-filled GitHub issue authored by the user — no API token ships in
  the binary (an embedded token would be extractable). Local queue keeps
  every submission for offline resilience.

## v1.9.2 — Premium display fix
- Position cards (Position Log + Sandbox) now show the position's total
  cash impact next to the per-unit entry: "collected $500" for credits,
  "paid $500" for debits (entryPrice × qty × multiplier). The close
  modal states the same total.

## Known limitations
- MAE/MFE only accumulate while the app is open (mark watermarks, not
  tick data).
- Watchlist CSV has no 52-week high/low columns (metric not fetched).
- Feedback screenshots are pasted into the GitHub issue by the user
  (issue URLs cannot carry images).
- Sharpe/Sortino/Calmar are per-trade-return based and unreliable on very
  short histories — the UI hints say so.

## Migration notes
- None required. First launch after update behaves normally; existing
  accounts never see the onboarding tutorial (fresh accounts only).
  All new preferences use new localStorage keys.

## Verification
- 126/126 backend tests; frontend `tsc` clean; production build clean.
- Every phase live-verified in the browser: onboarding flow end-to-end,
  analytics values checked against hand-computed results, export/clear/
  import round-trip, password re-key (old rejected, new accepted),
  account deletion, template save/load exactness, alert fire + dedup,
  feedback GitHub URL contents, premium math on all cards.
- All three themes (Obsidian, Emerald, Black & White) resolve correctly;
  all new UI rides the shared theme tokens.
